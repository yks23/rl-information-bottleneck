Finite Verifier Information and Dataset Geometry in RL

Why fixed-prompt RLVR saturates, and how transfer matrices reveal prompt interactions.

Poster PDF GitHub Data tables

Abstract

Fixed verifier datasets create a finite source of reward information. In mathematical RLVR, each prompt induces a binary verifier distribution whose local uncertainty is bounded by the prompt success probability. As training moves prompts toward all-correct or all-wrong regimes, useful verifier contrast can saturate. The practical ceiling is even tighter because prompt update directions are not fully aligned: information from one prompt does not automatically improve every other prompt. We therefore study the prompt-transfer matrix as a dataset geometry: it measures which prompts act independently, which prompts transfer, and how additive the dataset is under isolated RL updates.

Core Claims

Claim 1

Fixed verifier datasets have finite information.

For binary verifiers, prompt uncertainty is bounded by h2(p_i(t)) <= 1 and variance by p_i(t)(1 - p_i(t)) <= 1/4.

Claim 2

Weak cross-prompt transfer implies near-additivity.

If off-diagonal transfer is small compared with self-transfer, the dataset behaves like a collection of mostly separate information sources.

Claim 3

Transfer matrices define dataset geometry.

Training on source prompt i and evaluating target prompt j gives a directed interaction score that can be embedded, clustered, and used to study coverage.

Three Regimes, Unequal Weight

Traditional RL as contrast

AlphaZero-like self-play is a renewable source: the opponent and state distribution co-evolve with the learner. This keeps terminal outcomes near the decision boundary and prevents a fixed question list from being exhausted.

LLM RLVR as the main case

Fixed-prompt RLVR uses a non-renewable source. The main questions are how much verifier information is exposed, how much converts into performance, and how prompts interact through directed transfer.

Diffusion RL as a brief extension

The same lens can ask whether a diffusion reward source is finite and what geometry should be defined over denoising paths. This is an extension note, not a completed experimental claim.

Theory Sketch

Prompt uncertainty

H_i(t) = h2(p_i(t))

Verifier variance proxy

V_i(t) = p_i(t)(1 - p_i(t))

Dataset information exposure

A_D = sum_i sum_t H_i(t)

Directed prompt transfer

T_ij = p_j(after train i) - p_j(base)

The information bound alone is not the whole story. Prompt directions can disagree, so joint training cannot generally realize the sum of all prompt-wise ceilings. The transfer matrix measures this geometry directly.

Traditional RL Contrast

Self-play is the cleanest contrast to fixed-prompt RLVR. A terminal verifier can still be sparse and binary, but the source is renewable because the opponent and state distribution move with the current policy. This can keep reward variance close to the competence boundary rather than exhausting a fixed set of prompts.

Diffusion RL Note

Diffusion RL can be analyzed with the same source-and-geometry lens. The open problem is defining finite information and transfer geometry over denoising trajectories: which reward differences are aligned with controllable path directions, and when does the reward source saturate?
